全新视角下的 Java：一种非传统的免杀加载器轻松绕过杀软

引言

在信息安全领域中，加载器（Loader）始终扮演着关键角色，尤其在后渗透阶段。传统加载器多采用 C/C++ 编写，通过 Windows API 动态申请内存、写入 shellcode、创建线程执行。然而，在当前杀毒软件愈加敏感的趋势下，这些“套路化”的做法极易触发检测规则。

那么有没有一种“另辟蹊径”的方式？我选择尝试一个不被重视的方向 —— 使用 Java 实现 shellcode 加载与执行，并实测绕过主流杀软检测，包括 Windows Defender、360、火绒等。

这不是传统意义上的 Java 编程，而是借助 **JNA（Java Native Access）**技术，在 JVM 中直接调用底层 WinAPI，实现“免杀+执行”的新思路。

为什么选择 Java？

Java 在安全圈一直被视为“中规中矩的业务语言”，很少有人将其视为攻击链的一环。但恰恰因为它“太正常”，所以它具备一系列天然优势：

  • 字节码结构复杂，非标准 PE 文件，静态分析难度大；
  • jar 文件不包含 IAT 导入表，无传统 hook 点，天然规避许多检测；
  • 运行依赖 JVM，大多数沙箱不支持执行 jar 包；
  • 调用 API 方式非传统（通过 JNA/JNI），行为特征库中少有匹配；
  • 攻击面新颖，不易被蓝队规则捕捉；

这使得 Java 成为一个低特征、高执行力的绝佳 loader 平台。

核心实现思路：JNA 调用系统 API

Java 默认不能直接调用本地 API，但通过 JNA（Java Native Access），我们可以调用任意 DLL 中的函数 —— 无需 JNI，无需 C/C++ DLL，无需写 native 代码。

整个加载逻辑如下：

  1. 申请可执行内存：VirtualAlloc
  2. 写入 Shellcode 到内存：Pointer.write(...)
  3. 创建线程执行 Shellcode：CreateThread
  4. 等待线程结束：WaitForSingleObject

所有步骤均通过 JNA 实现，且全程不注入、无落地 EXE，无需外部 DLL。
