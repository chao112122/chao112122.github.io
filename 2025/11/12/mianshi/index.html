沥泉科技红队二面

  1. 你是明年（2026年）6月毕业吗？目前是大三？

  2. 你简历中提到参加过多次红队和蓝队行动，能否介绍一次由你主导或深度参与的渗透测试项目？

  3. 请简述该项目从前期信息收集 → 外部打点 → 内网横向移动 → 最终成果的完整流程。

  4. 你提到通过图书馆后台的 HTTPS 证书进行反查，具体是如何利用该证书定位到目标系统的,有过自己独特的信息收集技术吗？

  5. 后台管理员账号是如何获取的？是否通过“忘记密码”功能的逻辑漏洞实现邮箱爆破？该漏洞的具体机制是什么？

  6. 任意文件上传漏洞是否存在任何校验？上传后如何绕过 Windows Defender 等杀毒软件执行？

  7. 运维手册是通过什么方式获取的？

  8. 你提到通过运维手册联系工作人员申请了试用账号，该过程是否需要身份验证？试用系统开放了哪些权限？

  9. 你是如何在仅有学生端权限的情况下，通过代码审计发现教师端的 SQL 注入漏洞的？审计对象是源码还是黑盒测试？

  10. 该系统使用的是 MSSQL数据库，你是如何实现 OS 命令执行的？是否清楚SQLMAP的OS-Shell其原理？

  11. 除了 MSSQL，你还接触过哪些数据库（如 MySQL、SQL Server、达梦等）？请分别说明它们的常见利用条件和渗透思路。

  12. 对于 MySQL 的写文件（如 INTO OUTFILE），需要满足哪些前提条件（如权限、物理路径等）？

  13. 你提到 MySQL 的 UDF 提权，能否解释其原理？在真实环境中是否成功利用过？

  14. 如何判断目标数据库是否“不出网”（无法主动外连互联网）？在实战中是否遇到过“库网分离”的情况？如何绕过?

  15. 假设场景：你已获得数据库命令执行权限，但目标服务器不出网，如何将该主机“上线”（建立 C2 通信）？

  16. 了解过那些C2,你是否使用或搭建过 Cobalt Strike（CS）？是否了解其上线协议？

  17. 当目标不出网时，可通过哪些协议尝试上线？（如 DNS、SMB、ICMP 等）哪种协议在实际环境中成功率更高？

  18. 若使用 DNS 上线，如何将 payload 上传并执行于目标服务器？

  19. 你提到擅长免杀，请说明你的 payload 能绕过哪些杀软？（如 360、火绒、卡巴斯基、Defender 等）

  20. 针对动态行为检测（如命令执行、文件读写、注册表操作），你是如何规避的？

  21. 对于内存扫描（如卡巴斯基、Defender 的 AMSI 或内存扫描），你采用了哪些技术绕过,绕过内存扫描的原理是什么？（如 SEH 异常处理、自解密、内存加密等）

  22. 在流量侧，你是否对 C2 流量做过混淆或加密？使用的是自研 loader 还是修改现有框架？能否举例说明？

  23. 说一下 Windows 域环境和 Linux 内网

  24. 在域内渗透中，你常用的攻击路径或漏洞有哪些？（如 PrinterBug、PetitPotam、NTLM Relay、ACL 滥用等）

  25. 是否了解 LDAP 协议？它在内网信息收集中有哪些作用？

  26. 通过 LDAP 能够收集哪些域内信息？（如用户、组、OU、计算机、SPN 等）

  27. 如何识别域管理员（Domain Admin）或企业管理员（Enterprise Admin）账号？他们的关系是什么?

  28. 你是否使用过 BloodHound？对其流量特征和免杀问题有何认识？如果要在实战中使用，会如何改造以降低被发现风险？

  29. 你平时使用哪些代理工具？（如 frp、EarthWorm、Neo-reGeorg 等）

  30. 请解释正向代理与反向代理的区别，以及各自的适用场景。

  31. 使用 frp 配置正向代理时，是否需要在目标机开放端口？客户端如何连接仅有一个 IP/域名的目标？

  32. 在靶场或实战中，是否实际使用过 frp 等工具建立隧道？

  33. 你曾在wb实习，具体工作内容是什么？是否涉及红蓝对抗、应急响应或 APT 样本分析？

  34. 实习持续了多久？目前是否仍在校？

  35. 你目前所在地是广东，而我司位于四川成都。你是否接受长期异地工作？

  36. 我们的工作方向主要是对外情报收集（涉外目标，与国安/军方合作），与你此前做的甲方红队有所不同，你是否了解或有兴趣？

37.你在使用fscan等自动化工具时如何处理免杀 如何绕过EDR的流量检测

38.BloodHound如何免杀 在EDR下十分明显你要如何解决

39.有过vt和wb的沙箱上传经历吗 能做到全绿吗

40.过EDR的思路是什么

41.有改过c2的流量吗

深信服红队安研一面

  1. 请做简要自我介绍。

  2. 你目前的技术栈主要集中在哪些方向？（如外网渗透、免杀、内网、代码审计等）

  3. 你提到对外渗透和免杀比较熟悉，能否具体说明你在免杀方面主要使用哪些技术或工具？

  4. 你提到喜欢使用 C/C++ 编写 payload，能否说明你的免杀思路？（如加壳、加解密、冷门 API 调用、混淆等）

  5. 在反沙箱检测方面，你常用哪些手段？（如时间延迟、鼠标移动、硬盘/内存大小检测、概率性执行等）

  6. 你是否会使用反射式 DLL 注入或自行实现 Windows API？能否举例说明实现思路？

  7. 你提到“概率性执行”可绕过沙箱，能否解释其原理？（即沙箱通常只执行一次，而真实环境可多次触发）

  8. 你在外部打点中是否使用过 SQLMAP 等工具？SQLMAP的原理是什么是否有了解过？

  9. 你提到主要通过黑盒方式（如 TOP 漏洞、RCE、SSRF 等）进行渗透，能否举例说明你如何利用 SSRF？

  10. 对于反序列化类漏洞（如 Java RCE），你通常如何研究和复现？（如本地搭环境、动态调试、追踪调用链）

  11. 你提到熟悉 SQL 注入，若发现一个联合注入点，如何构造语句查询目标数据库的表结构和字段内容？

  12. 在利用 RCE 漏洞时，你如何判断目标是否“出网”？若出网，通常采用何种方式反弹 shell？（如 DNS 探测 + bash reverse shell）

  13. 你在wb实习期间同时参与红蓝对抗，使用过哪些安全产品？（如防火墙、EDR、蜜罐、WAF、SIEM 等）

  14. 在蓝队防守中，如何从海量告警中快速识别高价值攻击，排除低风险扫描行为？

    • 是否通过 IP 行为特征（如高频扫描、固定 payload）进行过滤？
    • 是否结合资产暴露面（如存在 Redis、Nexus 等服务）优先研判相关攻击？

  15. 对于工具类扫描流量（如目录爆破、信息泄露探测），你有哪些自动化或半自动化的过滤思路？

  16. 你是否分析过哥斯拉（Godzilla）、冰蝎（Behinder）、蚁剑（AntSword）等 WebShell 样本？

  17. 这些 WebShell 在流量特征上有何区别？

    • 例如：哥斯拉使用动态注册（如 Java 的 ClassLoader），流量中包含固定密钥和垃圾填充；
    • 冰蝎依赖 eval 或 assert 等函数，流量中可见特定函数调用。

  18. 你是否尝试过二开webshell以及流量

  19. 对于加壳或强混淆的样本，你如何进行动态/静态分析？是否具备独立分析能力

  20. 你是否使用 及如何用IDA、x64dbg 等工具进行恶意样本分析？

  21. 在分析一个钓鱼样本时，他的原理是什么 如何绕过的 你发现了哪些关键技术点？

    • 是否采用“白加黑”加载方式？
    • 是否通过注册表实现自启动？
    • 是否将自身目录加入 Windows Defender 排除列表？
    • 是否实现自定义 C2 协议（非 CS/Metasploit）？

  22. 该样本为独立的大马具备完整的远程控制功能（如文件上传下载、命令执行、内存保护）是什么原理？其执行逻辑如何组织？（如 switch 分发）

  23. 是否了解C2的架构原理,阅读过相关源码?

  24. 若想查找某个敏感 API（如 CreateRemoteThread、WinExec）的调用位置，你会如何定位？

  25. 对于样本分析,你是否能够完整进行动静态分析?

  26. 从原理上讲，为什么 IDA 等工具能解析 PE 文件的导入表？PE 结构中哪些字段用于存储导入函数信息？

  27. 你对未来工作岗位的期望是什么？（如专注渗透测试、红队行动等）

  28. 我们岗位偏安全研究（含二进制、漏洞挖掘、对抗技术），你是否有相关经验或兴趣？

  29. 你是否还有其他问题想了解？

  30. 对于国内系统的漏洞了解吗 有分析过吗 具体有哪些

  31. webshell的原理

  32. 对于之前分析的样本 他是怎么用一个维权手法
